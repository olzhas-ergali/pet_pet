Отчёт: бизнес-процесс, архитектура и запуск (OptBirja)

Статическая копия для печати. Актуальная версия: docs/BUSINESS-FLOW-REPORT.md в репозитории.

1. Запуск локально

  1. npm install
  2. Скопировать .env.example → .env, указать VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY.
  3. В Supabase выполнить миграции из supabase/migrations/ по порядку.
  4. Опционально: BFF, SUPABASE_SERVICE_ROLE_KEY для демо-пайплайна статусов.
  5. npm run dev — фронт.
  6. Docker: см. README.md, docs/LOCAL-POSTGRES.md.

2. Архитектура (кратко)

Слой Назначение
Vite + React SPA: лендинг, витрина, каталог, checkout, профиль, поставщик, админ.
Zustand Корзина, каталог, заказ, auth.
Supabase Auth, Postgres, RLS, RPC, Realtime.
BFF (Express) При VITE_USE_BFF=true: валидация корзины, интеграции.

3. Бизнес-процесс (целевой поток)

Клиент выбирает товар → оформление (submit_order) → события интеграций → при наличии BFF + service role демо-статусы paid → shipped → completed; поставщик видит заказы с его товарами; клиент видит трек в профиле.

4. Edge-кейсы

Пустая корзина, недостаток остатков, отсутствие цены, нет Supabase, отключение Realtime — см. подробности в Markdown-отчёте.

5. Ограничения

Отмена заказа без возврата остатков; демо-пайплайн синхронный; форма партнёра на лендинге без бэкенда; один Telegram chat в BFF.
